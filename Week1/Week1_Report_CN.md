# 周计划与进度报告（第1周）

## 一、第1周报告（2026-03-02 ~ 2026-03-06）

### 1. 达成的里程碑

1. 完成项目环境与模型接入验证  
   - 在 `langchain_test1.ipynb` 中完成 `ChatGoogleGenerativeAI` 接入与参数化翻译任务（PromptTemplate + style/text 变量）。
2. 完成基础数据准备与样本集生成  
   - 在 `data_preprocessing.ipynb` 中对评论数据进行读取、描述统计，并生成 `Reviews_top10percent.csv`（从 `Reviews.csv` 抽取前10%）。
3. 完成多种 LangChain 链路原型实现  
   - 在 `chain_prac1.ipynb` 中实现：
   - 普通链（Prompt -> LLM）
   - 顺序链（summary -> language -> follow_up）
   - 路由链（physics/math/history/computer_science 的 MultiPromptChain 路由）
4. 完成会话记忆能力验证  
   - 在 `memory_prac1.ipynb` 中实现 `ConversationBufferMemory`，验证多轮上下文保留与风格约束记忆。
5. 完成文档检索与向量索引原型  
   - 在 `doc_prac.ipynb` 中基于 `products.csv` 完成 `CSVLoader + VectorstoreIndexCreator + embeddings + Retrieval` 的查询流程。
6. 完成工具调用型 Agent 的初步实现  
   - 在 `test.py` 中实现 `add`、`multiply`、`get_weather`、`save_to_csv_with_path` 工具，并通过 `create_agent` 完成工具调用闭环。

### 2. 未达成的里程碑

1. 未完成端到端统一应用入口（当前仍分散在多个 notebook 与 `test.py`）。  
2. 未完成系统化评估（如摘要质量、检索准确率、路由准确率、工具调用成功率）。  
3. `get_weather` 仍为占位实现，未对接真实天气 API。

未达成原因：
- 本周主要投入在功能原型验证，优先保证“能力覆盖面”而非“工程整合”。
- 当前缺少统一评测数据与指标定义，导致评估模块尚未收敛。
- 外部 API 接入与异常处理策略尚未设计完成。

解决方案：
- 下周优先进行模块整合，形成统一入口脚本与标准化调用接口。
- 建立最小评测集与指标（准确率、可用率、耗时）并自动化记录结果。
- 将天气工具替换为真实 API 调用，补齐鉴权、超时、重试与错误处理。

### 3. 计划与工作量调整

- 2026-03-04（背景：已完成基础链路验证）  
  决策结果：将“高级优化”延后，先增加“Router Chain + RetrievalQA + Memory”覆盖，确保核心能力闭环可演示。

- 2026-03-06（背景：发现 notebook 分散，不利于汇报与复现）  
  决策结果：下周新增“工程整合”任务，统一输入输出接口与运行说明，提升可复现性与可维护性。

### 4. 可预见风险及应对

1. 风险：多模型/多链路组合后行为不稳定。  
   应对：固定关键参数（温度、提示词模板），建立回归测试样例。

2. 风险：外部 API 或密钥异常导致功能中断。  
   应对：增加 `.env` 校验、降级策略与错误提示。

3. 风险：数据规模增长导致检索与推理耗时上升。  
   应对：采用分批索引、缓存与更轻量模型进行对比测试。
